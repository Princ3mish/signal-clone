# Signal Clone

A pixel-perfect, full-stack Signal messenger clone. Built with **React 19 + TanStack Router** on the frontend and **FastAPI + SQLite** on the backend, connected via **REST API** for data and **WebSockets** for real-time messaging.

## Features

- **Real Authentication** — Register and login with bcrypt-hashed passwords + JWT tokens (7-day expiry)
- **Conversations** — Direct (1-on-1) and Group chats
- **Real-time Messaging** — WebSocket connection per open conversation
- **Optimistic UI** — Messages appear instantly with a temporary state, replaced by server-confirmed data
- **Typing Indicators** — Live "Alice is typing..." events via WebSocket
- **Read Receipts** — Messages transition through `sent → delivered → read` states with tick marks
- **Online Status** — Users appear online/offline in real time
- **Group Management** — Create groups, view members, admin controls
- **Contact System** — Users can see and start chats with contacts
- **Conversation Controls** — Archive, mute, delete chats via context menu
- **Settings Page** — Profile editing, theme toggle (light/dark), notification preferences
- **Responsive Layout** — Works on mobile and desktop with proper sidebar collapse

## Backend API Reference

All protected endpoints require the header:
```
Authorization: Bearer <jwt_token>
```

### Auth

| Method | Endpoint | Body | Response |
|---|---|---|---|
| POST | `/api/auth/register` | `{username, phone_number, display_name, password}` | `{user, token}` |
| POST | `/api/auth/login` | `{username, password}` | `{user, token}` |
| GET | `/api/auth/me` | — | `{user}` |

### Users

| Method | Endpoint | Notes |
|---|---|---|
| GET | `/api/users` | All users except self |
| GET | `/api/users/{id}` | Single user profile |
| PUT | `/api/users/{id}` | Update `display_name`, `avatar_url`, `phone_number` |

### Contacts

| Method | Endpoint | Notes |
|---|---|---|
| GET | `/api/contacts` | Your contact list |
| POST | `/api/contacts` | Add contact: `{contact_user_id}` |

### Conversations

| Method | Endpoint | Notes |
|---|---|---|
| GET | `/api/conversations` | All your conversations with members + unread count |
| POST | `/api/conversations` | Create direct: `{type:"direct", user_id}` or group: `{type:"group", name, member_ids:[]}` |
| GET | `/api/conversations/{id}` | Single conversation with members |
| PUT | `/api/conversations/{id}` | Update `group_name` or `group_avatar` |
| DELETE | `/api/conversations/{id}` | Removes you from the conversation |

### Messages

| Method | Endpoint | Notes |
|---|---|---|
| GET | `/api/conversations/{id}/messages` | Paginated: `?limit=50&offset=0` |
| POST | `/api/conversations/{id}/messages` | Send: `{content}` — broadcasts WS event |
| POST | `/api/messages/{id}/read` | Mark as read — broadcasts status update |
| DELETE | `/api/messages/{id}` | Delete own message |

### Group Members

| Method | Endpoint | Notes |
|---|---|---|
| POST | `/api/conversations/{id}/members` | Add member: `{user_id}` |
| DELETE | `/api/conversations/{id}/members/{user_id}` | Remove member (admin only) |

## WebSocket Events

**Connection URL:** `ws://localhost:8000/ws/{conversation_id}/{user_id}`

### Events you **receive** from the server

| Event type | Payload | When |
|---|---|---|
| `new_message` | `{message: {...}, sender_id: int}` | Someone sends a message |
| `message_status` | `{message_id, status}` | Message delivered/read |
| `user_typing` | `{user_id, conversation_id}` | Someone starts typing |
| `user_stopped_typing` | `{user_id, conversation_id}` | Someone stops typing |
| `user_online` | `{user_id}` | Someone connects |
| `user_offline` | `{user_id}` | Someone disconnects |

### Events you **send** to the server

| Event type | Payload | Effect |
|---|---|---|
| `typing_start` | `{conversation_id}` | Broadcasts `user_typing` to others |
| `typing_stop` | `{conversation_id}` | Broadcasts `user_stopped_typing` to others |

> **Self-echo prevention:** `new_message` events include `sender_id`. The frontend ignores events where `sender_id === currentUserId` since the sender already has the message from the API response.

## Frontend Architecture

### Routing (`src/routes/`)

TanStack Router uses **file-based routing**. Routes are auto-discovered and the route tree is generated in `routeTree.gen.ts`.

| Route | File | Description |
|---|---|---|
| `/` | `index.tsx` | Protected main app — redirects to `/login` if no auth |
| `/login` | `login.tsx` | Login + Register forms |
| `/register` | `register.tsx` | Alias redirect to `/login#register` |
| `/settings` | `settings.tsx` | Profile editing, theme, notifications |

### Data Flow

```
User opens conversation
        │
        ▼
ChatView useEffect fires
        │
        ├── useMessageStore.getState().fetchMessages(convId)
        │       └── GET /api/conversations/{id}/messages
        │           └── Sets messages[convId] in store
        │
        └── useMessageStore.getState().connectToConversation(convId, userId)
                └── Connects WebSocket ws://.../{convId}/{userId}
                └── Registers event handlers for new_message, message_status

User types message → send()
        │
        ├── Optimistic: adds temp message (id = negative timestamp) to store
        │
        ├── POST /api/conversations/{id}/messages
        │       └── Server saves to DB, broadcasts WS to other members
        │       └── Returns real message with server-assigned ID
        │
        └── Replaces temp message with real message in store
```

## State Management

All state lives in **Zustand** stores. `authStore` uses the `persist` middleware to survive page refreshes.

### `authStore`
```typescript
{
  currentUser: User | null
  token: string | null        // JWT — persisted to localStorage
  login(credentials)          // POST /api/auth/login
  register(data)              // POST /api/auth/register
  logout()                    // Clears user + token
  updateProfile(data)         // Updates currentUser locally
}
```

### `conversationStore`
```typescript
{
  conversations: Conversation[]
  users: User[]               // All known users (for contact/member display)
  activeConversationId: number | null
  fetchConversations()        // GET /api/conversations
  fetchUsers()                // GET /api/users
  createDirect(userId)        // POST /api/conversations {type:"direct"}
  createGroup(name, ids)      // POST /api/conversations {type:"group"}
  setActive(id)               // Sets active conversation
  updateConversation(id, patch)  // Local patch (mute, archive)
  markRead(id)                // POST /api/messages/{id}/read
  bumpToTop(id)               // Re-sort after new message
}
```

### `messageStore`
```typescript
{
  messages: Record<number, Message[]>  // keyed by conversationId
  fetchMessages(convId)        // GET /api/conversations/{id}/messages
  sendMessage(convId, content, senderId) // Optimistic + POST
  connectToConversation(convId, userId)  // Opens WebSocket + registers handlers
  sendTyping(convId, isTyping)           // Sends typing_start/stop via WS
  updateMessageStatus(msgId, status)     // Updates tick status in store
  deleteMessage(convId, msgId)           // Local removal
}
```

### `uiStore`
```typescript
{
  activeModal: string | null    // Which modal is open
  modalData: unknown            // Data passed to modal (e.g. groupId)
  typingUsers: Record<number, number[]>  // convId → [userId, ...]
  toastQueue: string[]          // Current toast messages
  openModal(name, data?)
  closeModal()
  showProfile(userId)
  toast(message)
  setTyping(convId, userIds)
}
```

## Auth Flow

```
REGISTER                           LOGIN
─────────────────────────────      ────────────────────────────────
Form: username, phone,             Form: username + password
      display_name, password              │
      │                                  ▼
      ▼                           POST /api/auth/login
POST /api/auth/register                  │
      │                           DB: SELECT user WHERE username=?
DB: INSERT INTO users                    │
    password → bcrypt.hashpw()    verify_password(plain, bcrypt_hash)
      │                                  │
      ▼                                  ▼
jwt.encode({sub: user_id,        jwt.encode({sub: user_id,
            exp: now+7days})                  exp: now+7days})
      │                                  │
      ▼                                  ▼
{ user, token } returned          { user, token } returned
      │                                  │
      └──────────────┬───────────────────┘
                     ▼
         authStore.set({ currentUser, token })
         Zustand persist → localStorage["signal-auth"]
                     │
                     ▼
         Every Axios request:
         Authorization: Bearer <token>
                     │
                     ▼
         FastAPI: current_user_id()
         → decode_token() → verifies signature + expiry
         → returns user_id (or 401)
```

**Security properties:**
- Passwords: bcrypt with random salt per user
- Tokens: HS256 JWT signed with `SECRET_KEY`
- Expiry: 7 days
- Transmission: `Authorization: Bearer` header only (not cookies)
- Storage: Zustand `persist` middleware → `localStorage`

## Setup & Running Locally

### Prerequisites
- **Node.js** 18+ (or use npm 10+)
- **Python** 3.11+

### 1. Clone & install frontend

```bash
cd beam-chat-hub-main
npm install
```

### 2. Set up backend

```bash
cd backend
pip install -r requirements.txt
```

### 3. Configure environment variables

**Frontend** — edit `.env` in project root:
```env
VITE_API_URL=http://localhost:8000
VITE_WS_URL=ws://localhost:8000
```

**Backend** — edit `backend/.env`:
```env
SECRET_KEY=change-this-to-a-random-secret
DATABASE_URL=./signal.db
CORS_ORIGINS=http://localhost:5173
```

### 4. Seed the database

```bash
cd backend
python seed.py
# Output: Seed complete!
```

This creates `signal.db` with 4 users, their contacts, 4 conversations, and 20 messages.

### 5. Start the backend

```bash
cd backend
python -m uvicorn main:app --reload --port 8000
```

### 6. Start the frontend

```bash
# From project root
npm run dev
```

Open **http://localhost:5173**

## Deployment

### Backend → Railway

1. Push `backend/` to a GitHub repo
2. Railway → **New Project** → **Deploy from GitHub Repo**
3. Set **Root Directory** to `backend`
4. Add environment variables:
   ```
   SECRET_KEY=<strong-random-string>
   CORS_ORIGINS=https://your-app.vercel.app
   DATABASE_URL=./signal.db
   ```
5. Set **Start Command**:
   ```
   uvicorn main:app --host 0.0.0.0 --port $PORT
   ```
6. After first deploy, open Railway shell and run:
   ```bash
   python seed.py
   ```

### Frontend → Vercel

1. Vercel → **New Project** → import repo
2. **Framework Preset**: Vite
3. **Build Command**: `npm run build`
4. **Output Directory**: `dist`
5. Add environment variables:
   ```
   VITE_API_URL=https://your-backend.railway.app
   VITE_WS_URL=wss://your-backend.railway.app
   ```
6. Deploy

> **Note:** SQLite is ephemeral on Railway's filesystem — for production persistence, replace with PostgreSQL (via `databases` + `asyncpg`) or attach a persistent volume.
